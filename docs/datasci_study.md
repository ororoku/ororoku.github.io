# データサイエンティストとしてやっていくには

必要なスキルのカテゴリーはざっくり３つある。
* ビジネス（課題の発見、起案や報告）
* エンジニアリング（プログラミング）
* サイエンス（数学、統計学）

スキルによっては複数にまたがる様なものも当然ある。
例えば経済学の知識の使い道は、ビジネスとサイエンスにまたがっていると思う。

また、本当にコアな能力は何かというと、結局、英数国の主要３教科になるのではないかと思う。
形式言語（数学）と自然言語（国語）の両方をちゃんと使えるということが全ての基礎であるし、英語はとりあえず出来るべきだろう。

以下、自分が有用だと思うスキルとその身に付け方について紹介する。

## 交渉や説得をやっていく
言うは易く行うは難しであるが、交渉の一般的なセオリーとしては以下の様なものがある
* ゼロサムゲームの場合
  * お互いのベストとワーストの重なる部分がどれくらいあるのか探っていく。ナッシュ交渉解。
* プラスサムゲームの場合
  * 全体のパイを大きくすることを強調する。win-winの関係
* どちらの場合でも重要なこと
  * 相手の根源的な要望（Why）に気を配る
  * 強力なプランBを用意しておく

説得においては、古典だがアリストテレスの弁論術（Ethos, Logos, Pathos, Metaphor, Brevity）が参考になるだろう。
[The Art of Persuasion Hasn’t Changed in 2,000 Years](https://hbr.org/2019/07/the-art-of-persuasion-hasnt-changed-in-2000-years)などを読んでみよう

## 社会科学のディシプリンを身に付ける
これは元物理屋として思うところであるが、理系の学問と、人間の意思決定が含まれるビジネスや社会の分析はやり方が根本的に異なる。
人間は粒子の様に押されればその通りに動く訳では無いので、重要な含意のある分析をしようと思えば、社会学なり経済学なり、何かしらの社会科学の体系が必要になる。
ただ、これは私がサービス業の分析をやっているからで、例えば新材料などの開発にデータサイエンスを使うなら、必要無いかもしれない。

社会人として知っておいて損は無いのは、初歩的な会計や法律の知識だろう。
